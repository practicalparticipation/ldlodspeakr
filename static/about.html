1. Overview

Linked Development is a prototype platform to connect up different datasets from research and practice in international development using Linked Open Data principles.

At present the platform hosts data from:

  • Research for Development (R4D) - with details of research publications and projects funded by the UK Department for International Development
  • ELDIS - an online information service hosted by the Institute of Development Studies providing free access to relevant, up-to-date and diverse research on international development issues.

The data on the platform is regularly updated from these sources. Further work to integrate additional sources is ongoing.

2. Data and data models

At present the majority of content hosted on Linked Development consists of meta-data on research publications.

This is drawn from a number of different sources, but is generally modelled using a model based on AGRIS RDF Properties, building on the recommendations from the LODE BD - Linked Open Data Enabled Bibliographic Data project.

Note on data models

Note that there may be slight differences between the models. In particular:

  • The two datasets use different categorisation schemes (represented as separate skos:ConceptSchemes) for categorising the subject and geographical coverage of resources. Mappings between these may be introduced in future.
  • The agrovoc and fao geography concept schemes used by R4D use skos:prefLabel and fao:nameList properties for titles, whereas ELDIS uses rdfs:label. A lite version of the Agrovoc vocabulary has been imported into the triple store, as have fao country profiles.
  • Creators (authors) are attached to R4D via blank nodes on the dcterms:creator property of a publication, but directly to ELDIS publications.

3. Interfaces

Linked Development is trialling a range of interface to facilitate access to Linked Data for international development. These include a SPARQL endpoint, Browse interface and an API based on the Institute for Development Studies open api.

Further details on how to use these can be found below.

4. SPARQL

A SPARQL endpoint can be found at /sparql, with a user-friendly SPARQL browser at /snorql.

SPARQL is a rich query language for Linked Data. Using SPARQL you can write queries to access any of the data held in the Linked Development data store. You can read more about SPARQL here.

The endpoint is provided by Virtuoso, and supports most SPARQL 1.1 features

5. API

The Linked Development API is a clone of the IDS API

It has been designed to allow applications written for the IDS API to be simply modified to talk to a large collection of documents and resources drawn from across the datasets including in Linked Development.

Accessing

The API is called using the following format: 

/openapi/{graph}/{function}/{object}/{parameter}/?{query}
where {graph} is one of 'all' (to search all available datasets), or 'eldis' or 'r4d' to restrict results to a given dataset.

For example /openapi/r4d/search/documents/short?country=ZA will fetch the short version of document records which record coverage of South Africa (ISO country code ZA).

Implemented features

  • Paging - using the ?num_results and ?start_offset URL paramters.
  • Response format selection - currently providing json and html.
  • Short or full results - substitute 'full' for 'short' in examples below to get full results.
A number of features are not yet fully implemented, including xml result formats, result counts (in some cases), and prev_page and next_page links. Please report any urgently required features on the GitHub issue tracker for the project.

Available services

get_all

  • Themes - e.g. /openapi/eldis/get_all/themes/short
  • Documents - e.g. /openapi/r4d/get_all/documents
  • Regions - e.g. /openapi/all/get_all/regions
  • Countries - e.g. /openapi/all/get_all/countries

search

The search service behaves similarly to the get_all endpoint, but with the option of adding search parameters for:

  • q - a free text query on titles and abstracts. E.g. /openapi/r4d/search/documents/full?q=linked data.
  • country - searching by ISO2 country code, or internal ELDIS identifier - e.g. /r4d/search/documents/full?country=GH.
  • Region - searching by UN code, or ELDIS identifier - e.g. /openapi/r4d/search/documents/full?region=UN002.
  • Theme - searching by name, agrovoc, or ELDIS identifier - e.g. /openapi/eldis/search/documents?theme=C782.

get

The get service is used for fetching individual resources by their ELDIS or R4D identifiers. For example: /openapi/eldis/get/documents/A64559/full or /openapi/r4d/get/documents/173629/full.

Additional features

IATI Identifier search

Taking advantage of information in the R4D dataset on the projects through which research is funded, and additional search parameter is available for documents. For example, /openapi/r4d/search/documents?iati-identifier=GB-1-114192&num_results=100 will fetch up to 100 documents associated with the DFID 'Programme for Enhancement of Research Information'

Linked Data URI

Responses include the Linked Data URI of entities included in the results in the linked_data_uri field.

Known issues

There are currently some performance issues with searches on the /all/ graph.

Source

The source code for the Linked Development API can be found on GitHub. This is also the place to report any bugs or to place feature requests.
